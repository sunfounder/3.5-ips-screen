FAQ
===========================

対応Raspberry Piモデル
-------------------------------------------

現在、この3.5インチディスプレイはRaspberry Pi 4およびRaspberry Pi 5のみに対応しています。Raspberry Pi 3B、3B+、およびZero 2Wでの使用は推奨されていません。タッチスクリーンドライバーのインストールにより、正常に動作しなくなる可能性があります。

このスクリーンはUbuntu、Kali Linux、またはRetroPieで使用できますか？
--------------------------------------------------------------------------------------

はい、可能ですが、特定のOSバージョンに限られます。  
未対応のバージョンを使用した場合、ドライバーインストール後にスクリーンが正常に動作しない可能性があります。

対応システム：

* **Raspberry Pi OS**

  - Trixie Desktop（64-bit）（リリース日：2025-12-04）
  - Bookworm Desktop（64-bit）（リリース日：2025-11-24、2025-05-13、2025-05-06）

    Bookwormにドライバーをインストールした後は、``startx`` を実行してデスクトップ環境を起動してください。

* **Ubuntu**

  - |link_ubuntu_2404| （Raspberry Pi 4のみ対応）

* **Kali Linux**

  - |link_kali_202503|

* **RetroPie**

  - |link_retropie| （Raspberry Pi 4のみ対応）

電源投入後、画面が黒または白のままになるのはなぜですか？
--------------------------------------------------------------------

* 正しいドライバーがインストールされているか確認してください（:ref:`install_driver`）。
* 使用しているOSが対応しているか確認してください（:ref:`install_os`）。
* スクリーンがGPIOピンに正しく接続されているか確認してください。

3.5インチIPSスクリーンとHDMIモニターを同時に使用できますか？
--------------------------------------------------------------------------------

いいえ。3.5インチIPSスクリーンドライバーをインストール後は、HDMIモニターを取り外すことを推奨します。

HDMIモニターを接続したままにすると、黒画面になる、または表示されない場合があります。

HDMI表示への復元
--------------------------

#. 3.5インチディスプレイドライバーのみをインストールし、config.txtに他の変更を加えていない場合は、以下のコマンドで復元できます：

   .. code-block:: bash

      cd LCD-show
      sudo ./system_restore.sh

   復元後、3.5インチディスプレイは使用できなくなり、再度ドライバーのインストールが必要になります。この方法は、比較的クリーンな環境で有効です。

#. config.txtも変更している場合は、手動での編集が最も確実です：

   * 以下のコマンドでconfig.txtを開きます：

     .. code-block:: bash

        sudo nano /boot/config.txt

   * ファイルの末尾までスクロールし、3.5インチディスプレイドライバーによる設定をコメントアウトします：

     .. code-block:: bash

        [all]
        hdmi_force_hotplug=1
        dtparam=i2c_arm=on
        dtparam=spi=on
        enable_uart=1
        dtoverlay=mhs35ips:rotate=90
        hdmi_group=2
        hdmi_mode=1
        hdmi_mode=87
        hdmi_cvt 480 320 60 6 0 0 0
        hdmi_drive=2

   * Ctrl+X → Y → Enter の順に押して保存し、終了します。

   * Raspberry Piを再起動すると、HDMI表示が復元されます。

   * 再度3.5インチディスプレイを使用する場合は、上記のコメントを解除してください。
   
インストール中に ``git not found`` やネットワークエラーが出た場合は？
--------------------------------------------------------------------------------------

Raspberry Piがインターネットに接続されていることを確認してください。

Gitがインストールされていない場合は、以下を実行してください：

* ``sudo apt update``
* ``sudo apt install git``

その後、再度インストールを実行してください。

画面の向きがおかしい場合は？
--------------------------------------------------------

以下のコマンドを実行することで、画面およびタッチの向きを修正できます：

* ``cd LCD-show/`` （使用しているOSに応じてパスを調整してください）
* ``sudo ./rotate.sh 90``

``90`` の代わりに ``0``、``180``、``270`` を指定して、希望の向きに変更できます。


ファンが常に回転しているのはなぜですか？ソフトウェアで制御できますか？
------------------------------------------------------------------------------------------

ファン用ヘッダーは常に5V電源を供給しており、ソフトウェアによる制御には対応していません。  
接続されている限り、ファンは常に回転し続けます。

動作温度範囲はどれくらいですか？
--------------------------------------------------------

対応する動作温度範囲は **-20°C～60°C** です。  
この範囲を超えると、故障や寿命の低下につながる可能性があります。

リモートアクセスのトラブル
----------------------------------

* **ssh: Could not resolve hostname ...**

  * ホスト名が正しいか確認してください。
  * 解決できない場合は、``<hostname>.local`` の代わりにIPアドレスを使用してください。

* **The term 'ssh' is not recognized...（Windows）**

  * OpenSSHがインストールされていません。手動でインストールしてください（:ref:`openssh_powershell` 参照）、  
    またはサードパーティ製のSSHクライアントを使用してください（:ref:`login_windows` 参照）。

* **Permission denied (publickey,password)**

  * Raspberry Pi Imagerで設定したユーザー名とパスワードが正しいことを確認してください。

* **Connection refused**

  * 電源投入後、1～2分待ってから再試行してください。
  * Raspberry Pi ImagerでSSHが有効になっていることを確認してください。
